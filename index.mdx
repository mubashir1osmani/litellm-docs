# LiteLLM - Getting Started

[GitHub](https://github.com/BerriAI/litellm)

## **Call 100+ LLMs using the OpenAI Input/Output Format**

- Translate inputs to provider's `completion`, `embedding`, and `image_generation` endpoints
- [Consistent output](https://docs.litellm.ai/docs/supported_endpoints) — same response format regardless of provider
- Retry/fallback logic across multiple deployments (e.g. Azure/OpenAI) — [Router](/routing)
- Track spend & set budgets per project — [LiteLLM Proxy Server](/proxy/docker_quick_start)

## How to use LiteLLM

You can use LiteLLM through either the **Proxy Server** or **Python SDK**. Both give you a unified interface to 100+ LLMs.

| | **LiteLLM Proxy Server** | **LiteLLM Python SDK** |
|---|---|---|
| **Use Case** | Central service (LLM Gateway) to access multiple LLMs | Use LiteLLM directly in your Python code |
| **Who Uses It?** | Gen AI Enablement / ML Platform Teams | Developers building LLM projects |
| **Key Features** | Centralized API gateway with auth & authorization, multi-tenant cost tracking, virtual keys, admin dashboard | Direct Python integration, router with retry/fallback, load balancing, OpenAI-compatible errors, observability callbacks |

## **LiteLLM Proxy Server (LLM Gateway)**

Track spend across multiple projects, set budgets, and access 100+ LLMs through a single API.

### Quick Start

```shell
pip install 'litellm[proxy]'
litellm --model huggingface/bigcode/starcoder

# RUNNING on http://0.0.0.0:4000
```

<Note>
  The proxy is also available as a [Docker image](/proxy/docker_quick_start).
</Note>

### Test the proxy

```shell
curl http://0.0.0.0:4000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-3.5-turbo",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```
